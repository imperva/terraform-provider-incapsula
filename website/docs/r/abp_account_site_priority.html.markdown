# incapsula_abp_account_site_priority

Manages the priority order of Sites within an ABP Account. The first ID in
`site_ids` has the highest priority. The list must include every Site
attached to the Account exactly once.

NOTE: this resource wraps a singleton property of the parent Account.
Destroying it removes the priority from Terraform state but leaves the
server-side order unchanged.

## Example Usage

```terraform
# Defines the order in which an account's Sites are matched. Sites are matched
# top-down. Both Terraform-managed sites and externally-managed sites (looked up
# via the `incapsula_abp_site` data source) can be referenced.
resource "incapsula_abp_account_site_priority" "accprio" {
  account_id = var.account_id
  site_ids = [
    incapsula_abp_site.sample_site.id,
    incapsula_abp_site.site2.id,
    data.incapsula_abp_site.external_site.id,
  ]
}
```
